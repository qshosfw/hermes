# <CheckCircle2 className="inline w-6 h-6 mr-2 text-emerald-400" /> 2. Type 0: Acknowledgment (ACK)

When a packet requests an Acknowledgment (`Want Ack` = 1), the receiving node responds rapidly with a **Type 0** Packet to definitively confirm reception (or explicitly request mesh retransmission).

> [!IMPORTANT]
> ACKs are exclusively utilized during Unicast communication. Due to extreme bandwidth limitations, Multicast groups rarely request network-wide ACKs.

## 9.1 Base ACK Structure

The ACK payload completely utilizes the 56-byte envelope constraint to securely sign for the target packet while simultaneously supplying highly compressed backchannel data.

```mermaid

        A["Packet ID<br/>(6B)"] --- B["Inner MAC<br/>(8B)"] --- C["Bitfield<br/>(1B)"] --- D["Health<br/>(5B)"] --- E["Location<br/>(17B)"]
    end
    
    classDef data fill:transparent,stroke:#89b4fa,stroke-width:2px,color:inherit
    classDef opt fill:transparent,stroke:#a855f7,stroke-width:2px,stroke-dasharray: 4 4,color:inherit
    
    class A,B,C data
    class D,E opt
```

| Field | Range (Bytes) | Description |
|---:|---:|---|
| **ACKed Packet ID** | `0 - 5` (6 Bytes) | The unique 48-bit identifier of the packet being ACKed. |
| **Inner MAC** | `6 - 13` (8 Bytes) | The static e2e Poly1305 signature of the subject packet's payload. |
| **Bitfields** | `14` (1 Byte) | Contains routing statuses, fragment definitions, and telemetry toggles. |
| **Health Blob** | `15 - 19` (5 Bytes) *(Optional)* | Active if the Telemetry bit is `1`. RSSI metrics, LQI, and Battery Voltage. |
| **Location Blob** | `20 - 36` (17 Bytes) *(Optional)* | Active if the Telemetry bit is `1`. Packed GPS latitude, longitude, and heading data. |

## 9.2 The Bitfields (Byte 14)

Byte 14 densely packs flags crucial to evaluating the payload state and adjusting Transmit backoff timing.

| Bits | Use | Value / Options |
|:---:|:---|:---|
| **4** (`<< 4`) | Fragment Index | `0-15`. Specifies exactly which subset piece is being acknowledged. |
| **1** (`<< 3`) | Last Fragment | `1` indicates that the sender acknowledges the completion of the physical chain. |
| **2** (`<< 1`) | ACK Status | `0` = OK (No Errors) · `1` = OK (Corrected via FEC) · `2` = NACK (Unrecoverable, No Retry) · `3` = NACK (Please Retransmit) |
| **1** (`& 0x01`) | Telemetry Bit | `1` informs the receiver that bytes 15-36 contain the packed Health and Location Blobs rather than 0-padded nulls. |

## 9.3 The Telemetry Blobs

If the payload Transport specifies the Telemetry bit, an ACK leverages its remaining bytes to supply incredibly dense statistics directly referencing the connection parameters.

### Fast Integer Sigmoid LQI

The Link Quality Indicator (LQI) provides a robust `0-255` metric evaluating the probability of transmission success. Rather than employing computationally expensive floating-point math (e.g., Sigmoids with exponentiation), Hermes implements a **Fast Integer Sigmoid** (Elliot sigmoid approximation) optimized perfectly for bit-shift operations on embedded cores.

LQI evaluates the Signal-to-Noise Ratio (SNR) for the Forward (ACKed) and Reverse (ACKing) paths. SNR is derived by subtracting the `Idle RSSI` ($R_{idle}$) from the target packet RSSI ($R_{pkt}$ or $R_{ack}$):

```math
S_fwd = R_pkt - R_idle \quad \text{and} \quad S_rev = R_ack - R_idle
```

For each path, we evaluate an integer representation of the Link Probability curve. Centered at a Midpoint of $M=6$ and Scale of $K=2$:

```math
P_int(S) = 127 + \frac {(S - M) * 128} {|S - M| + K}
```

The final `0-255` integer representation transmitted is the combined average of the forward and reverse paths, safely clamped to byte boundaries:

```math
LQI_fast = \text{clamp}(0, \frac {P_int(S_fwd) + P_int(S_rev)} {2}, 255)
```

> [!TIP]
> **Embedded Optimization:** Multiplication by `128` is heavily optimized inside embedded ARM Cortex chips directly as a left bit-shift operator (`<< 7`), while `|x|` maps to `abs()`. This approach flawlessly maps an S-curve logistic regression spanning `0` to `255` utilizing strict integer division avoiding all floating-point operations.

### The RSSI Triad

Every ACK Health Blob carries **three distinct RSSI measurements** to characterize the bidirectional link:

| Metric | Description | Purpose |
|:---|:---|:---|
| **ACKed RSSI** ($R_{pkt}$) | Signal strength of the data packet that is being acknowledged. | Forward path quality. |
| **ACKing RSSI** ($R_{ack}$) | Signal strength of the *previous* packet received from the destination node. | Reverse path quality. |
| **Idle RSSI** ($R_{idle}$) | Ambient noise floor measured by the radio just before reception. | Establishes the SNR baseline. |

By combining all three values, the sender gains a complete picture of the **bidirectional SNR** without requiring a separate diagnostic exchange.

### 9.3.1 Health Blob (5 Bytes)

| Byte Address | Bit Allocation | Data Field | Range / Note |
|:---:|:---:|:---|:---|
| `15` | **1** (`0x80`) | Has Battery | Defines if following 7 bits hold voltage data. |
| `15` | **7** (`0x7F`) | Battery Voltage | 0.0V–12.7V in 0.1V steps. |
| `16` | **7** | ACKed RSSI | Received signal strength of the packet being ACKed (-120 to +6 dBm). |
| `16 - 17` | **7** | ACKing RSSI | Signal strength of the previous packet from the destination. |
| `17 - 18` | **6** | Idle RSSI | Background noise floor metrics directly logged from the radio receiver (-120 to -60 dBm). |
| `18 - 19` | **8** | Previous LQI | LQI scale (`0-255`) utilizing the fast integer sigmoid evaluating link reliability. |
| `19` | **4** | TX Power | Represents the active Logarithmic Power scale Output from the Radio hardware. |

### 9.3.2 Location Blob (17 Bytes)

Utilizing the remaining 17 bytes of the ACK envelope perfectly, nodes attach their current global mapping parameters (if previously enabled inside network privacy configuration settings).

The location blob compresses decimal strings heavily:

| Metric | Bit Length | Allowed Range | Transformation Formula |
|:---|:---:|:---|:---|
| **Latitude** | `24 Bits` | ±90 degrees | `(Lat + 90) × (16777215 / 180)` |
| **Longitude** | `24 Bits` | ±180 degrees | `(Lon + 180) × (16777215 / 360)` |
| **Altitude** | `20 Bits` | -1000m to +30,000m | `(Alt + 1000) × 10` |
| **GPS Week** | `10 Bits` | 0 to 1023 | Direct mapping |
| **Time of Week** | `20 Bits` | 0 to 604,799 s | Direct mapping |
| **Speed** | `12 Bits` | 0.00 to 40.95 m/s | `Speed × 100` |
| **Heading** | `12 Bits` | 0.0 to 359.9° | `Heading × 10` |
| **Satellites** | `6 Bits` | 0 to 63 | Direct mapping |
| **Precision** | `8 Bits` | 0.0m to 25.5m | `HDOP × 10` |
