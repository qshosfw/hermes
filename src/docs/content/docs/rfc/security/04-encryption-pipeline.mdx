# <Zap className="inline w-6 h-6 mr-2 text-yellow-400" /> 4. Encryption Pipeline

To understand Hermes security, we must trace a 56-byte message through its transformation from plaintext to radio waves.

## 4.1 Step 1: End-to-End AEAD (Inner Layer)
**Goal**: Privacy between sender and recipient.

1. The sender generates a **96-bit Random Nonce**.
2. The sender derives $K_{scope}$ using $K_{mesh}$ and the **Destination ID**.
3. The **56-byte Payload** and the **Source ID** are encrypted using **ChaCha20-Poly1305**.
4. **Result**: A 64-byte cipher block containing the encrypted payload and an 8-byte authentication tag.

## 4.2 Step 2: Mesh Obfuscation (Outer Layer)
**Goal**: Hide routing metadata and signatures from passive listeners.

1. The sender (or router) generates a **32-bit Hop Nonce**.
2. A **Mesh Keystream** is generated using $K_{mesh}$ and the **Hop Nonce**.
3. The **Header** (excluding the Hop Nonce itself) and the **Inner Cipher block** are XORed with this keystream.
4. **Result**: A 96-byte obfuscated block where every individual bit appears random.

## 4.3 Step 3: LFSR Whitening (Physical Layer)
**Goal**: Spectral integrity and DC bias elimination for the radio hardware.

1. The 96-byte packet is XORed with a **PN15 LFSR sequence** (x^15 + x^14 + 1).
2. **Result**: Final bitstream ready for FSK modulation.
