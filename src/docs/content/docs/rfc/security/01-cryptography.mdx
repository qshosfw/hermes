# <Lock className="inline w-6 h-6 mr-2 text-blue-400" /> 1. Cryptographic Primitives

Hermes Link relies on the **ChaCha20-Poly1305** Authenticated Encryption with Associated Data (AEAD) construct. These primitives were selected for their high software performance on embedded 32-bit MCUs (like the ARM Cortex-M series).

## 1.1 ChaCha20 Stream Cipher

ChaCha20 is a stream cipher that generates a pseudo-random keystream by XORing plain-text with the output of its internal 20-round permutation function.

- **Key Size**: 256 bits (32 bytes).
- **Nonce Size**: 96 bits (12 bytes).
- **Counter**: 32 bits (4 bytes).

## 1.2 Poly1305 MAC

Poly1305 is a high-speed Message Authentication Code (MAC) that provides cryptographic proof that the data (and its associated metadata) has not been tampered with.

- **Output size**: 128 bits (16 bytes).
- **Truncation**: In Hermes, the MAC is truncated to **64 bits** (8 bytes) for the inner/outer split signatures.

## 1.3 Keys & Secrets

Hermes uses a tiered key structure to balance shared access and individual privacy.

| Key | Symbol | Scope | Derive From |
| :--- | :--- | :--- | :--- |
| **Network Key** | $K_{net}$ | Mesh-wide | User Passcode |
| **Mesh Secret** | $K_{mesh}$ | Single Mesh | $K_{net}$ + Global Salt |
| **Traffic Key** | $K_{scope}$ | Pair or Group | $K_{mesh}$ + Target ID |

- **$K_{mesh}$ (The Chameleon)**: Obfuscates all packets on the air. Every node in the network knows this.
- **$K_{scope}$ (Personal)**: Encrypts the actual message payload and the sender's identity. Only the intended recipient can derive this.

## 1.4 Nonces & Variation

Nonces ensure that the same message never looks the same twice on the air, preventing replay and pattern analysis.

| Nonce | Bits | Purpose | Changes When? |
| :--- | :--- | :--- | :--- |
| **Hop Nonce** | 32 | Scrambles outer layer | Every single hop (re-randomized) |
| **Packet Nonce**| 96 | AEAD Inner Nonce | Generated by sender per packet |

## 2. Encryption Pipeline: What is hidden?

Hermes performs "Sealed Sender" encryption, where even the routing metadata is mostly hidden from passive observers.

| Visibility | Field | State | Why? |
| :--- | :--- | :--- | :--- |
| 🟢 **Clear** | Destination ID | Obfuscated | Routers need this to forward. |
| 🟢 **Clear** | TTL & Flags | Obfuscated | Routers need to detect loops. |
| 🔴 **Hidden** | **Source ID** | **Encrypted** | Privacy. Routers don't need to know who sent it. |
| 🔴 **Hidden** | **Payload** | **Encrypted** | Confidentiality. |
| 🔴 **Hidden** | Inner Signatures| **Encrypted** | Prevents metadata fingerprinting. |

> [!NOTE]
> **"Obfuscated" vs "Encrypted"**
> In Hermes, even "cleartext" fields are XORed with a keystream derived from $K_{mesh}$ and the Hop Nonce. To a passive listener, **every bit** of the packet appears as random noise. Only nodes with the Mesh Key can "de-shroud" the routing header.

## 1.4 Why not AES-GCM?

Unlike AES, which is slow and vulnerable to cache-timing attacks on many low-cost microcontrollers without dedicated hardware acceleration (AES-NI), **ChaCha20** is inherently resistant to such attacks and offers significantly better performance in software-only environments.

> [!TIP]
> **Performance Metric**
> On a 48MHz ARM Cortex-M0+ (like the one in many radios), ChaCha20-Poly1305 can process a 64-byte block in less than **4,000 clock cycles**, which is comfortably within the timing requirements for real-time mesh routing.
