# 9. Type 0: Acknowledgment (ACK)

When a packet requests an Acknowledgment (`Want Ack` = 1), the receiving node responds rapidly with a **Type 0** Packet to definitively confirm reception (or explicitly request mesh retransmission).

> **Important:** ACKs are exclusively utilized during Unicast communication. Due to extreme bandwidth limitations, Multicast groups rarely request network-wide ACKs.

## 9.1 Base ACK Structure

The ACK payload completely utilizes the 54-byte envelope constraint to securely sign for the target packet while simultaneously supplying highly compressed backchannel data. 

```mermaid

    A["ACKed Nonce<br>(12 Bytes)"] --- B["ACKed Signature<br>(16 Bytes)"] --- C["Bitfields<br>(1 Byte)"] --- D["Health Blob<br>(5 Bytes)"] --- E["Location Blob<br>(17 Bytes)"]
    classDef default fill:#1e1e2e,stroke:#89b4fa,stroke-width:2px,color:#ffffff;
```

| Field | Range (Bytes) | Description |
|---:|---:|---|
| **ACKed Nonce** | `0 - 11` (12 Bytes) | The exact 96-bit Nonce from the Transport Header of the Packet being ACKed. |
| **ACKed Signature** | `12 - 27` (16 Bytes) | The exact 128-bit Poly1305 Signature of the Packet being ACKed. |
| **Bitfields** | `28` (1 Byte) | Contains routing statuses, fragment definitions, and telemetry toggles. |
| **Health Blob** | `29 - 33` (5 Bytes) *(Optional)* | Active if the Telemetry bit is `1`. RSSI metrics, LQI, and Battery Voltage. |
| **Location Blob** | `34 - 50` (17 Bytes) *(Optional)* | Active if the Telemetry bit is `1`. Packed GPS latitude, longitude, and heading data. |

## 9.2 The Bitfields (Byte 28)

Byte 28 densely packs flags crucial to evaluating the payload state and adjusting Transmit backoff timing.

| Bits | Use | Value / Options |
|:---:|:---|:---|
| **4** (`<< 4`) | Fragment Index | `0-15`. Specifies exactly which subset piece is being acknowledged. |
| **1** (`<< 3`) | Last Fragment | `1` indicates that the sender acknowledges the completion of the physical chain. |
| **2** (`<< 1`) | ACK Status | `0` = OK (No Errors)<br/>`1` = OK (Corrected via FEC)<br/>`2` = NACK (Unrecoverable, No Retry)<br/>`3` = NACK (Please Retransmit) |
| **1** (`& 0x01`) | Telemetry Bit | `1` informs the receiver that bytes 29-50 contain the packed Health and Location Blobs rather than 0-padded nulls. |

## 9.3 The Telemetry Blobs

If the payload Transport specifies the Telemetry bit, an ACK leverages its remaining 25 bytes to supply incredibly dense statistics directly referencing the connection parameters. 

The Link Quality Indicator (LQI) provides a robust `0-255` metric evaluating the probability of transmission success. Rather than employing computationally expensive floating-point math (e.g., Sigmoids with exponentiation), Hermes implements a **Fast Integer Sigmoid** (Elliot sigmoid approximation) optimized perfectly for bit-shift operations on embedded cores.

LQI evaluates the Signal-to-Noise Ratio (SNR) for the Forward (ACKed) and Reverse (ACKing) paths. SNR is derived by subtracting the `Idle RSSI` ($R_{idle}$) from the target packet RSSI ($R_{pkt}$ or $R_{ack}$):

```math
S_{fwd} = R_{pkt} - R_{idle} \quad \text{and} \quad S_{rev} = R_{ack} - R_{idle}
```

For each path, we evaluate an integer representation of the Link Probability curve. Centered at a Midpoint of $M=6$ and Scale of $K=2$:

```math
P_{int}(S) = 127 + \frac{(S - M) \ll 7}{|S - M| + K}
```

The final `0-255` integer representation transmitted is the combined average of the forward and reverse paths, safely clamped to byte boundaries:

```math
LQI_{fast} = clamp\left(0, \frac{P_{int}(S_{fwd}) + P_{int}(S_{rev})}{2}, 255\right)
```

> **Embedded Tip:** Multiplication by `128` is heavily optimized inside embedded ARM Cortex chips directly as a left bit-shift operator (`<< 7`), while `|x|` maps to `abs()`. This approach flawlessly maps an S-curve logistic regression spanning `0` to `255` utilizing strict integer division avoiding all floating-point operations.




### 9.3.1 Health Blob (5 Bytes)

| Byte Address | Bit Allocation | Data Field | Range / Note |
|:---:|:---:|:---|:---|
| `29` | **1** (`0x80`) | Has Battery | Defines if following 7 bits hold voltage data. |
| `29` | **7** (`0x7F`) | Battery Voltage | 0.0V–12.7V in 0.1V steps. |
| `30` | **7** | ACKed RSSI | The received signal strength of the packet being ACKed (-120 to +6 dBm). |
| `30 - 31` | **7** | ACKing RSSI | Signal strength of the previous packet from the destination. |
| `31 - 32` | **6** | Idle RSSI | Background noise floor metrics directly logged from the radio receiver (-120 to -60 dBm). |
| `32 - 33` | **8** | Previous LQI | LQI scale (`0-255`) utilizing the fast integer sigmoid evaluating link reliability. |
| `33` | **4**  | TX Power | Represents the active Logarithmic Power scale Output from the Radio hardware. |

### 9.3.2 Location Blob (17 Bytes)

Utilizing the remaining 17 bytes of the ACK envelope perfectly, nodes attach their current global mapping parameters (if previously enabled inside network privacy configuration settings). 

The location blob compresses decimal strings heavily:

- **Latitude**: `24 Bits` (`-90` to `+90`)
- **Longitude**: `24 Bits` (`-180` to `+180`)
- **Altitude**: `20 Bits` (`-1000m` to `+30,000m`)
- **GPS Week Time**: `10 Bits` 
- **Time Of Week**: `20 Bits` (Seconds 0 - 604,799)
- **Speed**: `12 Bits` (m/s)
- **Heading**: `12 Bits` (Degrees 0.0 - 359.9)
- **Satellites**: `6 Bits` 
- **Precision (Accuracy Radius)**: `8 Bits` (meters) 

Nodes parsing the 17 raw bytes reconstruct these values through bit-shift division.
